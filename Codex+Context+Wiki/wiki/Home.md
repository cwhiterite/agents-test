# Software Design Principles and Patterns Wiki

This wiki is a practical decision aid for software design work with an AI assistant. It combines enduring object-oriented principles, design pattern catalogs, architecture patterns, distributed systems patterns, and a project-context folder model for agent-assisted delivery.

Use it to:

- Frame design discussions before code is written.
- Compare alternatives with trade-offs instead of pattern shopping.
- Give AI assistants stable project rules and architecture context.
- Record decisions, progress, and design rationale across a project.

## Core Rule

Patterns are vocabulary, not goals. Start with forces: coupling, cohesion, volatility, performance, consistency, operability, team ownership, user experience, and change frequency. Choose the simplest design that satisfies the forces you can actually name.
